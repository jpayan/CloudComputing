  1. Read the AWS Well Architected Framework and make a summary about the 5 pillars https://d0.awsstatic.com/whitepapers/architecture/AWS_WellArchitected_Framework.pdf

    There are always many ways of accomplishing an objective, most of them focus on specific aspects, and among these ways we can also always find the right way of doing something. What differences the right way from the others is planning, designing taking into consideration every single requirement of the objective to be accomplished. And the vast world of Computer Science is not excluded from the previous statement, being the objective a strong, reliable, consistent, longevous system. Hence, here are the five pillars of the Well-Architected Framework.

    • Security Pillar

      Security means being able to protect information and the system itself from any kind of corruption, regardless of its origin (external or internal). It is of prime importance to establish rules concerning privileges of components and/or users, to have a fine-grained defined access policy, and to respond accordingly to any type of incident.
      In order to achieve and ensure a system's security, here are a few main-focus points:

      • The system should posses a multi-level security, protecting the flow of data between every layer, instead of doing so for only the layer that communicates the system to the exterior.
      • Errors must be traceable anywhere in the system. To do so, all actions should be logged.
      • Avoiding an overly permissive system by handling each one of the system's interaction with only the necessary permissions.
      • Ensuring the integrity of the data / avoiding corruption.
      • Creating a trust zone by automating the response of the system to any malfunction or uncommon behavior.
      The best practices to fulfill the previous focus points are:
      Identity and Access Management: Making sure that only authorized users are able to interact with the system's resources, and that resources have no possibility of interacting with resources in an unintended way.
      Detective Controls: Constant monitoring of the system, analysis of logs and implementation of alarms that allows the developers to tract and isolate unwanted behaviors.
      Infrastructure Protection: Boundary protections, enforcing multiple level security.
      Data Protection: Categorizing data by sensitivity and encrypting primordial data.
      Incident Response: Accordingly and automatically respond to any incident that happens within the system.

    • Reliability Pillar

      Having a reliable system means having a system that is able to recover from any incident and autoscale depending on its demand. To achieve this, we must focus on testing not only the behavior of the system in specific situations, but also recovery strategies and automating them; breaking down the system into smaller components to add modular maintenance; adding an autoscaling capacity to the system, in order to avoid over saturation of resources; automating every change that wants to be done to the system.

    • Performance Efficiency Pillar

      An efficient system is that one that correctly adapts to the requirements and keeping the performance constant in many situations. To keep a good performance in the system, we should utilize the adequate technologies; deliver accessibility to the system in multiple parts of the planet to minimize latency.

    • Cost Optimization Pillar

      Minimizing costs for a system implies clinging to the absolutely necessary technical specifications, and improving this with time. A few good practices to minimize costs without affecting the system's performance are avoiding having resources running while not being used; use a cloud computing provider to eliminate the need of physical equipment besides working stations.

    • Operational Excellence Pillar

      The operational excellence is achieved by performing a well planned development process, in which every part of the system is tested against unexpected behaviors, environments and events; by allowing easy and quick understanding of the code through good documentation and code reviews; by implementing a modular design that allows the development team to maintain the system with small changes rather than changes that imply modifying most of the system itself; by improving based on previous events or situations that were not expected and caused malfunctioning of the system and by automating most of the previously mentioned.
